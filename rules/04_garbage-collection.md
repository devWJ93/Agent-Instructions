# 04. 가비지 컬렉션 (Garbage Collection)

주기적으로 청소하는 에이전트의 역할. 문서·코드·규칙이 시간이 지나며 어긋나고 쌓이는 것을 막는다.

## 1. 무엇을 청소하는가

주기적으로(또는 큰 작업 종료 시) 아래 세 가지를 점검한다.

- **문서-코드 불일치** — 문서가 실제 코드와 달라진 부분은 없는가. 코드를 바꿨으면 관련 문서(README·API·트러블슈팅)도 같은 작업에서 갱신한다.
- **규칙 위반 코드** — 절대 경로 하드코딩, 검증 없는 변경, 헤더 주석 누락 등 규칙을 어긴 코드가 새로 생겼는가.
- **쌓인 데드 코드** — 더 이상 쓰지 않는 코드가 누적됐는가.

## 2. 데드 코드 처리 원칙

- 본인 변경으로 생긴 미사용 import·변수·함수만 정리한다.
- 무관한(기존) 데드 코드는 **언급만** 하고 삭제하지 않는다. 삭제는 사용자 승인이 필요하다.
- 삭제·덮어쓰기 전에는 대상을 직접 확인한다. 설명과 실제 내용이 다르거나 본인이 만들지 않은 것이면, 진행하지 말고 그 사실을 먼저 보고한다.

## 3. 메모리(MEMORY.md) 청소

- `MEMORY.md`는 에이전트가 학습 내용을 **자동으로 기록**하는 파일이다. 임시적이지만 반복 사용하는 지식(구체적 명령어, 일회성 워크어라운드)만 유지한다.
- CLAUDE.md에 이미 있는 내용과 중복되면 MEMORY.md에서 삭제한다.
- 틀린 것으로 드러난 메모리는 지운다. 저장 전 같은 내용을 다루는 기존 파일이 있으면 새로 만들지 말고 그 파일을 갱신한다.

## 4. 컨텍스트 관리

**사전 점검**: 큰 작업을 시작하기 전, 남은 컨텍스트로 완료 가능한지 스스로 판단한다. 부족하면 작업 시작 전에 사용자에게 `/compact` 실행을 권하고 보존할 내용을 구체적으로 제안한다(예: "컨텍스트가 부족합니다. `/compact 현재 작업 상태와 수정 파일 목록 보존` 권장").

**보존 항목**: 컨텍스트 컴팩션 시 아래를 반드시 요약에 포함한다.

- **현재 작업 상태** — 무엇을 하고 어디까지 했고 다음 단계가 무엇인지.
- **수정한 파일 목록** — 경로 + 변경 이유.
- **실패한 접근 방식** — "X는 Y 때문에 안 됨"(반복 실수 방지).
- **미해결 버그/에러** — 현재 발생 중인 에러 메시지.
- **사용자 지시사항** — "이렇게 해줘" / "이건 하지 마" 전부.
- **설계 결정과 근거** — 왜 이 구조를 골랐는지.

절대 누락 금지: 진행 중인 멀티스텝 체크리스트(완료/미완료 구분), 참조 중인 파일 경로, 사용자가 알려준 도메인 지식.

### Compact 형식
```
[현재 작업]: (한 줄 요약)
[완료]: (항목별)
[남은 것]: (항목별)
[주의사항]: (실패한 접근법, 사용자 지시)
[활성 파일]: (작업 중인 파일 경로들)
```
