# Security Policy

이 저장소는 공개용 운영 지침과 도구 카탈로그만 담는다. 실제 인증 정보, 개인 설정, 비공개 프로젝트 지식은 포함하지 않는다.

## 커밋 금지 항목

다음 파일과 값은 저장소에 커밋하지 않는다.

- API key, access token, refresh token, OAuth client secret.
- `.env`, `.env.local`, `.pem`, `.key`, private key, 인증 헤더 원문.
- 개인 계정의 로컬 설정 파일.
- 내부망 IP, 사설 저장소 주소, 비공개 프로젝트 경로.
- 세션 로그, 에이전트 trace, 채팅 원문처럼 프롬프트와 경로가 섞인 파일.

## 허용 항목

다음 형태는 공개 가능하다.

- 환경변수 이름 예시. 예: `HF_TOKEN`, `QODO_API_KEY`.
- 마스킹된 토큰 예시. 예: `<YOUR_TOKEN>`.
- 공개 문서나 공식 MCP 서버 연결 방법.
- 직접 작성한 운영 지침과 카탈로그 설명.

## 공개 전 점검 명령

PowerShell에서 저장소 루트 기준으로 실행한다.

```powershell
rg -n --hidden -g '!.git/**' "(ntn_[A-Za-z0-9_\-]+|ghp_[A-Za-z0-9_]+|github_pat_[A-Za-z0-9_]+|sk-[A-Za-z0-9]{20,}|BEGIN [A-Z ]*PRIVATE KEY|AKIA[0-9A-Z]{16}|C:\\Users\\[^\\]+|192\.168\.)"
```

출력되는 항목이 실제 비밀정보나 개인 환경 정보라면 커밋 전에 제거하거나 일반화한다. 문서에서 환경변수 이름만 설명하는 항목은 허용할 수 있다.

## 보안 이슈 처리

민감정보가 이미 공개 저장소에 올라갔다면 다음 순서로 처리한다.

1. 해당 토큰이나 키를 즉시 폐기하고 재발급한다.
2. 공개 커밋에서 값을 제거한다.
3. 필요한 경우 GitHub secret scanning 경고를 확인한다.
4. 재발 방지를 위해 `SECURITY.md`와 관련 지침을 보강한다.

기록에 남은 토큰은 삭제 커밋만으로 안전해지지 않는다. 값이 한 번 공개됐다면 반드시 회전한다.
