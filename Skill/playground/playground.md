# playground

> 컨트롤·라이브 프리뷰·프롬프트 출력을 갖춘 자기완결형 단일 HTML 인터랙티브 탐색기를 만드는 스킬

## 📌 용도
대화형 HTML 플레이그라운드를 만든다. 한쪽에 인터랙티브 컨트롤, 다른 쪽에 라이브 프리뷰, 하단에 복사 버튼이 달린 프롬프트 출력이 있는 자기완결형 HTML 파일이다. 사용자가 컨트롤을 조정해 시각적으로 탐색한 뒤, 생성된 프롬프트를 복사해 Claude에 다시 붙여넣는다. 입력 공간이 크거나 시각적·구조적이어서 일반 텍스트로 표현하기 어려울 때 특히 유용하다.

## 🎯 트리거 조건
- **언제 호출되는가**: 플레이그라운드, 탐색기(explorer), 특정 주제용 대화형 도구 제작 요청 시.
- **언제 쓰지 않는가**: 일반 프로덕션 프론트엔드 UI(→ frontend-design). 플레이그라운드는 "설정→프리뷰→프롬프트 복사" 패턴 전용.

## 💻 사용 예시 / 명령어
- "카드 디자인을 조절해보는 플레이그라운드를 만들어줘"
- "SQL 쿼리 빌더 탐색기를 만들어줘"
- "이 개념을 시각적으로 탐색하는 인터랙티브 도구"

## 🛠️ 제공 도구 / 주요 파라미터
**사용 방법**:
1. 요청에서 플레이그라운드 유형 식별.
2. `templates/`에서 매칭 템플릿 로드:
   - `design-playground.md` — 시각 디자인 결정(컴포넌트, 레이아웃, 간격, 색상, 타이포)
   - `data-explorer.md` — 데이터·쿼리 빌딩(SQL, API, 파이프라인, 정규식)
   - `concept-map.md` — 학습·탐색(개념 맵, 지식 격차, 범위 매핑)
   - `document-critique.md` — 문서 리뷰(승인/거부/코멘트 워크플로)
   - `diff-review.md` — 코드 리뷰(git diff, 커밋, PR 라인별 코멘트)
   - `code-map.md` — 코드베이스 아키텍처(컴포넌트 관계, 데이터 흐름, 레이어 다이어그램)
3. 템플릿을 따라 빌드(맞는 게 없으면 가장 가까운 것 적응).
4. 작성 후 `open <filename>.html`로 브라우저 실행.

**핵심 요구사항(모든 플레이그라운드)**:
- **단일 HTML 파일**: CSS·JS 인라인, 외부 의존성 없음.
- **라이브 프리뷰**: 모든 컨트롤 변경 시 즉시 갱신("Apply" 버튼 없음).
- **프롬프트 출력**: 값 덤프가 아닌 자연어, 비기본값만 언급, 플레이그라운드 없이도 실행 가능한 컨텍스트 포함, 라이브 갱신.
- **복사 버튼**: 클립보드 복사 + "Copied!" 피드백.
- **기본값 + 프리셋**: 첫 로드부터 보기 좋게, 3~5개 명명된 프리셋.
- **다크 테마**: UI는 시스템 폰트, 코드/값은 모노스페이스, 최소 chrome.

**상태 관리 패턴**: 단일 state 객체 — 모든 컨트롤이 쓰고, 모든 렌더가 읽음. 각 컨트롤이 `updateAll()`(renderPreview + updatePrompt) 호출.

**흔한 실수**: 프롬프트가 값 덤프(→ 자연어 지시), 컨트롤 과다(→ 관심사별 그룹화·고급 접기), 즉시 갱신 안 됨, 기본값/프리셋 없음, 외부 의존성, 프롬프트 컨텍스트 부족.

**참조 파일**: `templates/` 디렉터리(6종 템플릿)
