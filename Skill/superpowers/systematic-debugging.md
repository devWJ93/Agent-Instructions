# systematic-debugging

> 어떤 버그·테스트 실패·예상 밖 동작이든, 수정안을 내기 전에 반드시 근본 원인을 먼저 찾는 4단계 체계적 디버깅 스킬.

## 📌 용도
무작위 수정은 시간을 낭비하고 새 버그를 만든다. 임시 패치는 근본 문제를 가린다. 핵심 원칙은 "수정 시도 전 항상 근본 원인을 찾는다. 증상 수정은 실패다"이다. 철의 법칙: **근본 원인 조사 없이는 어떤 수정도 금지**(Phase 1 미완료 시 수정 제안 불가). 4단계(근본 원인 조사 → 패턴 분석 → 가설·테스트 → 구현)를 순서대로, 각 단계를 완료해야 다음으로 넘어간다.

## 🎯 트리거 조건
- **언제 호출되는가**: 모든 기술적 이슈 — 테스트 실패, 프로덕션 버그, 예상 밖 동작, 성능 문제, 빌드 실패, 통합 이슈. **특히** 시간 압박, "딱 한 번만 빠른 수정"이 유혹적일 때, 이미 여러 수정 시도, 이전 수정 실패, 이슈를 완전히 이해 못 할 때.
- **언제 쓰지 않는가(건너뛰지 말 것)**: "단순해 보임"(단순 버그도 근본 원인 있음), "급함"(서두르면 재작업 보장), "지금 당장 고치라"(체계적이 더 빠름) — 모두 건너뛰면 안 되는 경우.
- **스킬 유형**: Rigid — 철의 법칙과 단계 강제.

## 💻 사용 예시 / 호출 방법
4단계:
- **Phase 1 근본 원인 조사**: ①에러 메시지 정독(스택트레이스·라인·코드 완독) ②일관 재현(신뢰성 있게 트리거되나) ③최근 변경 확인(git diff, 새 의존성/설정) ④다중 컴포넌트 시스템이면 경계마다 진단 계측 추가(입력/출력/환경전파/상태 로깅) 후 1회 실행해 "어디서" 깨지는지 증거 수집 ⑤데이터 흐름 추적(나쁜 값의 발원지까지 거슬러 — `root-cause-tracing.md`).
- **Phase 2 패턴 분석**: 작동하는 유사 예제 찾기, 참조 구현 완독(대충 읽지 말 것), 차이점 전부 나열("그게 중요할 리 없어" 금지), 의존성 이해.
- **Phase 3 가설·테스트**: 단일 가설 명시("Y 때문에 X가 근본 원인") → 최소 변경으로 테스트(한 번에 한 변수) → 검증 후 진행, 실패 시 새 가설(수정 위에 수정 추가 금지) → 모르면 "X를 모르겠다" 솔직히.
- **Phase 4 구현**: 실패 테스트 케이스 먼저 작성(`test-driven-development` 사용) → 단일 수정(한 번에 하나, "온 김에" 개선 금지) → 검증 → 안 되면 STOP·시도 횟수 세기(<3이면 Phase 1 복귀, **≥3이면 아키텍처 의문**).

## 🛠️ 제공 내용 / 핵심 절차
- **3회 이상 실패 시 아키텍처 의문**: 매 수정이 다른 곳에서 새 공유상태/결합 문제 노출, 수정에 "대규모 리팩토링" 필요, 매 수정이 다른 증상 유발 → 패턴이 근본적으로 타당한가, 관성으로 버티는가, 리팩토링 vs 증상 수정. 더 수정하기 전 파트너와 논의. (실패한 가설이 아니라 잘못된 아키텍처.)
- **Red Flags(STOP·Phase 1 복귀)**: "일단 빠른 수정 나중 조사", "X 바꿔보고 보자", "여러 변경 후 테스트", "테스트 건너뛰고 수동 검증", "아마 X일 거야", "이해 못 하지만 될 수도", 추적 전 해법 제안, "한 번만 더 시도"(이미 2회+), 매 수정이 새 문제 노출.
- **파트너의 경고 신호**: "그거 안 일어나?"(검증 없이 가정), "보여줄까?"(증거 수집 필요), "추측 그만"(이해 없이 수정 제안), "Ultrathink this"(근본 의문), "막혔어?"(접근이 안 통함).
- **흔한 합리화 반박표**, **Quick Reference 표**(Phase별 활동·성공기준) 포함.
- **보조 기법**: `root-cause-tracing.md`(역추적), `defense-in-depth.md`(다층 검증), `condition-based-waiting.md`(임의 타임아웃→조건 폴링). 연계: `test-driven-development`(실패 테스트), `verification-before-completion`(수정 검증).
- "근본 원인 없음"의 95%는 불완전한 조사다.
